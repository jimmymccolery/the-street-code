## The pattern named

A drafting-drift fire happens when the drafting substrate (operator or AI assistance) produces text against memory of canonical state rather than against an immediate canonical-state read. The drafted content references stale, fabricated, or compressed canonical content; canonical state read at execution time reveals the mismatch.

The original framing, canonized at research_queue.md line 865 (Topic 20 description): "Operator memory of canonical state is unreliable even during revision work explicitly addressing canonical-state-drift family patterns. Drafting revision text against memory rather than against immediate canonical-state read produces drafting-drift fires — proposed edits referencing fictional or stale canonical content. Discipline implication: verify-canonical-first applies to revision drafting, not just initial writing; reading canonical state immediately before drafting any specific edit is operationally load-bearing."

The mechanism distinguishes drafting-drift from adjacent failure modes. Anti-pattern #10 covers verify-before-stating: substrate fails to verify before producing claims about canonical state. Drafting-drift operates one layer down: substrate has access to canonical state but does not read it before drafting text whose content depends on it. Memory-of-canonical-state and canonical-state-read are distinct verification modalities; treating them as equivalent produces the drift.

The discipline countermeasure, per Topic 20 preliminary rule: verify-canonical-first applies recursively across drafting sites. Reading canonical state immediately before any drafting site whose content depends on canonical state is operationally load-bearing. Memory of canonical state from earlier in the same session is unreliable even when the recall is recent.

## What the discipline catches and what it costs

What single-pass drafting plus single-pass review misses: drafted text that resembles canonical content but is not byte-equal. Memory-of-canonical-state produces approximations close enough to read past on first pass. The drift becomes visible only when text is verified against canonical state, either by Claude Code's str_replace old_str matching constraint (structural check at tooling layer), by Phase 5 review (separate verification pass), or by external audit (cross-substrate or operator-side direct grep).

What it costs: enforcement attention at every drafting site. The discipline is per-edit, per-find-string, per-citation, per-content-claim. Bulk drafting cannot batch the verification cleanly because each drafted segment may reference different canonical content. The cost compounds in citation-heavy work; banked post #4's six fires within roughly ten citations is empirical evidence that the per-site cost is real, and that batching to Phase 5 review alone does not catch what per-section verification catches.

What it buys: drafted artifacts that match canonical state at byte-equal precision. Phase 5 batched review's surface area shrinks; correction overhead per fire drops; cumulative draft accuracy improves. The cost-benefit calibration is operator-judgment territory; at high citation density or high stakes (canonical-record mutations; published methodology research; canonical anti-pattern promotion arcs), the discipline pays for itself by reducing downstream correction work.
