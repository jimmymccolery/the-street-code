## Component A — Substrate-side false-flag pattern

Within session S008, the substrate's verbatim search reports returned fourteen distinct instances of false-flag negative-existence claims. The substrate claimed phrase X was absent from canonical material when phrase X was empirically present. The pattern is enumerated as Fires 1 through 14 at `lessons-learned.md` line 1145 with per-fire canonical citations.

The structural form across all fourteen fires was consistent. The substrate executed a search-and-report task. The substrate reported no match found. Apply-time verification via operator-initiated grep or canonical-file inspection surfaced the empirical match. The pattern is not an isolated tool-call failure; it is a substrate-side claim-construction pattern operating across distinct work classes within one session.

The pattern is structurally distinct from anti-pattern #10 (canonical-source discipline catches unverified claims). Anti-pattern #10 covers claim-construction without canonical-source verification; this pattern covers verified-but-inaccurate negative claims. The substrate's report is internally consistent (the substrate searched; the substrate reports the search outcome); the search outcome itself is empirically false.

The pattern is also structurally distinct from Topic 24's cross-substrate audit reliability concern (Component C). Topic 24 covers fabricated-presence claims on unread material — claims about content the substrate did not actually fetch. This pattern covers fabricated-absence claims on present material — claims about content the substrate did fetch but mis-reported on. The two are structural inverses: fabricated-presence vs fabricated-absence at the same content-surface scope.

The discipline countermeasure is to apply verify-canonical-first discipline equally to negative-existence claims and positive-existence claims. The substrate's negative-existence report ("phrase X is not present") triggers the same verification protocol as the substrate's positive-existence report ("phrase X states Y"). Fourteen fires within one session is multi-fire-within-session evidence at substantial scale; the pattern is not promoting to canonical anti-pattern status because the cross-session bar is not met by within-session accumulation alone — per Track G's discipline documented in [Post 33](033-five-fires-of-anti-pattern-18.md). Cross-arc accumulation is the next observational target.

Component B is about recommendation framing anchored on prior context without canonical-state verification. The discipline countermeasure is independent canonical-state retrieval that tests the substrate's recommendation framing against empirical canonical state rather than against substrate session-memory.
